FIG 0 End-to-end composition

Let configuration and taxonomy be fixed inputs. The orchestrator implements a pipeline that maps corpus documents to static HTML and JSON artifacts:

\[ \Pi \;=\; \mathrm{Report}\;\circ\;\mathrm{Serialize}\;\circ\;\mathrm{Compare}\;\circ\;\bigl(\mathrm{GT},\,\mathrm{Pred}\bigr)\;\circ\;\mathrm{Ingest}. \]

Here Pred is either an LLM-backed extractor, a deterministic stub, or rows loaded from agent assessments (same row schema). GT loads expert-coded rows per document.

FIG 1 Ingestion as a structured mapping

For document map mode, let \(\mathcal{M}\) be the set of map entries. Each entry yields one pipeline document \(d\) with identifiers and texts read from disk:

\[ \mathrm{Ingest}(\mathcal{M}) \;=\; \bigl\{\,\phi(i) : i \in \mathcal{M}\,\bigr\},\quad \phi(i) = \bigl(\mathrm{id}_i,\;\mathrm{RUS}_i,\;\mathrm{ENG}_i,\ldots\bigr). \]
  • \(\mathrm{RUS}_i\) is loaded from data/russian_originals/{id}.txt (after sync from dev).
  • \(\mathrm{ENG}_i\) tries data/input/{id}.txt, spaced variants, display-name paths, optional en_filename, then derives filenames from the Russian source filename under dev/english_translations so CI clones still get full English.

The segmentation surface for extraction is primarily \(\mathrm{RUS}_i\) (Russian-first); English full text is used for bilingual display and corpus-level English token statistics.

FIG 2 Extraction: model as conditional mapping

Let \(\mathcal{C}\) and \(\mathcal{F}\) be finite category and framing label sets from taxonomy \(T\). A row lives in:

\[ r \;\in\; \mathcal{S} \;=\; \mathrm{Section}\times\Sigma^\ast\times\Sigma^\ast\times\mathcal{C}\times\mathcal{F}\times\Sigma^\ast. \]

Stub (no live LLM): split \(\mathrm{RUS}_i\) into short phrases \(p_1,\ldots,p_K\) by newlines and sentence-like breaks; keep \(p_j\) with length \(\ge 4\), cap \(K\le 25\). Labels cycle through taxonomy lists:

\[ \mathrm{Pred}_{\mathrm{stub}}(i) \;=\; \Bigl\{\,\bigl(j,\,\varepsilon,\,p_j,\,c_{j \bmod m},\,f_{j \bmod n},\,p_j\bigr)\,\Bigr\}_{j=1}^K,\quad m=|\mathcal{C}|,\; n=|\mathcal{F}|. \]

Ollama: build prompt \(P_i\) containing truncated Russian document \(\mathrm{truncate}(\mathrm{RUS}_i,8000)\) and enumerated allowed labels. The model implements a stochastic map; the adapter parses JSON lines into \(\mathcal{S}\):

\[ \mathrm{Pred}_{\mathrm{LLM}}(i) \;=\; \mathrm{Parse}\bigl(\mathcal{A}_\theta(P_i)\bigr),\quad P_i = \mathrm{Prompt}\bigl(\mathrm{truncate}(\mathrm{RUS}_i,\,8000),\,\mathcal{C},\,\mathcal{F}\bigr). \]

If the request fails or parsing yields nothing, the pipeline falls back to \(\mathrm{Pred}_{\mathrm{stub}}(i)\).

Agent assessments: \(\mathrm{Pred}_{\mathrm{agent}}(i)\) is read directly from JSON keyed by \(\mathrm{id}_i\). Rows must match the same schema so downstream \(\mathrm{Compare}\) is unchanged.

FIG 3 Ground truth

Expert annotations yield an ordered list \(\mathrm{GT}(i) = (g_1,\ldots,g_{n_i}) \in \mathcal{S}^{n_i}\) with the same schema as \(\mathrm{Pred}\). Loading normalizes labels to canonical taxonomy strings for storage and comparison.

\[ \mathrm{GT}\colon \mathrm{id}\;\mapsto\;(g_1,\ldots,g_{n_i}),\qquad g_k\in\mathcal{S}. \]

FIG 4 Alignment as bipartite pairing

Let \(L=\mathrm{Pred}(i)\) and \(G=\mathrm{GT}(i)\) with \(|G|=n_H\), \(|L|=n_L\). Normalize phrase text:

\[ \eta(s) \;=\; \mathrm{lower}\bigl(\text{collapse-ws}(\mathrm{strip}(s))\bigr). \]

Content alignment (default content_rus): bipartite edges when \(\eta(\texttt{entry\_rus})\) matches exactly or one normalized string contains the other (minimum length 4). Greedy passes assign each GT row at most one LLM partner.

\[ \mathcal{P}_i \;\subseteq\;\{1,\ldots,n_H\}\times\{1,\ldots,n_L\},\quad |\mathcal{P}_i| \le \min(n_H,n_L). \]

Index alignment (match_by: index): rows pair by position \(k\) up to \(\max(n_H,n_L)\).

FIG 5 Accuracy metrics

\[ \mathrm{cat}(r)=\mathbf{1}\{\texttt{llm\_category}=\texttt{human\_category}\},\quad \mathrm{fram}(r)=\mathbf{1}\{\texttt{llm\_framing}=\texttt{human\_framing}\}. \]

Let \(\mathcal{R}^{\mathrm{p}}_i\) be paired rows. Percentages use only paired rows:

\[ \mathrm{Acc}^{\mathrm{cat}}_i \;=\; \frac{100}{|\mathcal{R}^{\mathrm{p}}_i|}\sum_{r\in\mathcal{R}^{\mathrm{p}}_i}\mathrm{cat}(r),\quad \mathrm{Acc}^{\mathrm{fram}}_i \;=\; \frac{100}{|\mathcal{R}^{\mathrm{p}}_i|}\sum_{r\in\mathcal{R}^{\mathrm{p}}_i}\mathrm{fram}(r), \]
\[ \mathrm{Acc}^{\mathrm{both}}_i \;=\; \frac{100}{|\mathcal{R}^{\mathrm{p}}_i|}\sum_{r\in\mathcal{R}^{\mathrm{p}}_i}\mathrm{cat}(r)\cdot\mathrm{fram}(r). \]

If \(|\mathcal{R}^{\mathrm{p}}_i|=0\), all three reported percentages are \(0\).

Under index alignment, denominator \(|\mathcal{A}_i|=\max(n_H,n_L)\).

\[ \mathrm{Acc}^{\mathrm{cat}}_i \;=\; \frac{100}{|\mathcal{A}_i|}\sum_{r\in\mathcal{A}_i}\mathrm{cat}(r). \]

FIG 6 Report and bilingual highlighting

The serializer writes \(\{\texttt{documents},\texttt{comparison\_by\_doc}\}\) to JSON. Document panels show \(\mathrm{ENG}_i\) and \(\mathrm{RUS}_i\) with spans at \(\texttt{entry\_eng}\) / \(\texttt{entry\_rus}\). Repeated phrases consume occurrences in row order; overlapping spans resolve with shorter segments preferred after sorting by \((\mathrm{length},\mathrm{index})\).

\[ \mathrm{Report}\bigl(\mathrm{Compare}(\mathrm{Pred},\mathrm{GT}),\,\mathrm{Ingest}(\mathcal{M}),\,T\bigr)\;\to\;\mathrm{HTML}. \]

L1 · HTTP interface (Ollama)

Fix hyperparameters \(\theta = (\texttt{model\_id},\,\tau,\,M,\,t_{\mathrm{out}})\): model name, sampling temperature \(\tau\), maximum predicted tokens \(M\), HTTP timeout. The adapter sends a JSON body whose semantic content is the pair \((\texttt{model},\,P_i,\,\texttt{options})\) with \(\texttt{options}=\{\texttt{temperature}:\tau,\texttt{num\_predict}:M\}\).

\[ \mathrm{HTTP}_\theta \colon \Sigma^\ast \to \Sigma^\ast,\quad \mathrm{HTTP}_\theta(P_i) \in \Sigma^\ast \cup \{\bot\}, \]

where \(\bot\) denotes transport failure, HTTP error, or empty decode — formally collapsing all failures so \(\mathrm{Parse}(\bot)=\emptyset\) by convention.

L2 · Truncation & prompt functional

Let \(X_i=\mathrm{RUS}_i\) be the Russian document string from ingest. Define prefix truncation \(\pi_k\colon \Sigma^\ast\to\Sigma^\ast\) keeping the first \(k\) characters (implementation uses \(k=8000\)):

\[ \pi_k(X) = X_{[1:k]}. \]

The prompt functional \(\mathrm{Prompt}\colon \Sigma^\ast \times 2^{\mathcal{C}} \times 2^{\mathcal{F}} \to \Sigma^\ast\) concatenates instructions, enumerates \(\mathcal{C}\) and \(\mathcal{F}\), and embeds \(\pi_k(X_i)\). Thus

\[ P_i := \mathrm{Prompt}(\pi_k(X_i),\mathcal{C},\mathcal{F}). \]

L3 · End-to-end extraction map

The neural generator \(\mathcal{A}_\theta\) is a stochastic kernel; after fixing randomness it becomes a function \(\Sigma^\ast\to\Sigma^\ast\). The adapter extracts the textual field response from JSON and applies deterministic parsing:

\[ \mathrm{Pred}_{\mathrm{LLM}}(i) \;=\; \mathrm{Parse}\Bigl(\mathrm{Sel}_{\texttt{response}}\bigl(\mathrm{HTTP}_\theta(P_i)\bigr)\Bigr), \]

where \(\mathrm{Sel}_{\texttt{response}}\) is undefined on failure (yielding parseable empty input). Section L7 factorizes the stochastic map inside \(\mathrm{HTTP}_\theta\) into tokenization, a causal language model, and sampling.

L4 · Parse & label coercion

\(\mathrm{Parse}\) scans lines of the model output; each line that decodes as JSON dict contributes at most one row. Missing category/framing strings are replaced by the first taxonomy element when absent from \(\mathcal{C}\) or \(\mathcal{F}\).

Proposition L4.1. Every row emitted by \(\mathrm{Parse}\) satisfies \(\texttt{content\_category}\in\mathcal{C}\) and \(\texttt{framing}\in\mathcal{F}\) after coercion.

Proof. The implementation tests membership and, on failure, assigns \(c_0\in\mathcal{C}\) or \(f_0\in\mathcal{F}\) as first list entries (or empty string only if lists empty — taxonomy loader guarantees nonempty lists in practice). \(\square\)

Remark. Coercion is not semantically neutral: arbitrary strings collapse to default labels, which can inflate superficial agreement unless models obey the closed vocabulary.

L5 · Total predictive closure

Theorem L5.1. Define the pipeline predictor \(\mathrm{Pred}^\ast(i)\) by

\[ \mathrm{Pred}^\ast(i) \;=\; \begin{cases} \mathrm{Pred}_{\mathrm{LLM}}(i) & \text{if Ollama mode is enabled and } \mathrm{Pred}_{\mathrm{LLM}}(i)\neq\emptyset,\\ \mathrm{Pred}_{\mathrm{stub}}(i) & \text{otherwise (fixture mode or fallback).} \end{cases} \]
Proof. Matches llm/__init__.py: on exceptions or empty parsed rows, control assigns \(\mathrm{Pred}_{\mathrm{stub}}(i)\). The stub always returns a nonempty finite list (including a degenerate row when no phrase qualifies). \(\square\)

L6 · Stub as partition-like segmentation

Let \(\mathrm{Split}\colon \Sigma^\ast \to (\Sigma^\ast)^\ast\) be the deterministic splitting rule (newline + sentence-dot heuristic). Let \(\mathrm{Filter}_{\ge \ell}\) remove strings shorter than \(\ell=4\). Let \(\mathrm{Take}_{\le K}\) keep the first \(K=25\) survivors.

\[ (p_1,\ldots,p_{K'}) \;=\; \mathrm{Take}_{\le K}\bigl(\mathrm{Filter}_{\ge \ell}(\mathrm{Split}(X_i))\bigr),\quad K'\le K. \]

Lemma L6.1. \(\mathrm{Pred}_{\mathrm{stub}}(i)\) has length \(\le 25\) and each phrase \(p_j\) has \(\lvert p_j\rvert \ge 4\) unless the degenerate “no text” row fires.

Proof. Immediate from guards MIN_PHRASE_LEN and slice [:25] in llm/__init__.py. \(\square\)

L7 · Neural generator: prompts → text assessments

This abstracts what happens after the Python adapter posts prompt to Ollama’s /api/generate with stream: false. The identifier model_id selects a frozen parameter vector \(\theta\) (e.g. Llama-class decoder-only transformer weights loaded by the server). Our pipeline treats the server as implementing one draw from a conditional distribution over continuation strings.

L7.1 · Deployment architecture

The assessment string \(Y_i\in\Sigma^\ast\) is produced outside the Vozmezdie process boundary: the client only sends \(P_i\) and decoding knobs \((\tau,M)\); the server runs kernel schedules, KV-cache updates, and sampling until a stopping rule fires.

Fig L7a — Request body matches ollama_adapter.py: stream: false waits for the full continuation before returning one JSON payload.

L7.2 · Discrete channel: tokens & vocabulary

Let \(\mathcal{V}\) be the model’s finite vocabulary (including specials such as BOS, EOS, padding). The server-side tokenizer implements

\[ \mathrm{Tok}\colon \Sigma^\ast \to \mathcal{V}^L,\qquad L=L(P_i), \]

a deterministic map from Unicode prompt text to a length-\(L\) token sequence \(z_{1:L}\). Let \(\mathrm{Detok}\colon \mathcal{V}^\ast\to\Sigma^\ast\) invert fragments by concatenating decoded pieces (implementation-defined joins). The prompt occupies positions \(1,\ldots,L\); generation appends \(z_{L+1},z_{L+2},\ldots\).

Fig L7b — Decoder-only models apply causal masking so position \(t\) never attends to \(t{+}1,\ldots\); the hidden \(h_t\) at the final prompt index feeds the language-model head.

L7.3 · Autoregressive assessment distribution

Conditioned on prompt tokens \(z_{1:L}\), the model defines a distribution over continuations by chaining conditional categoricals. Writing \(\tilde{z}_{1:t}=(z_1,\ldots,z_t)\) for the growing prefix,

\[ p_\theta\bigl(z_{L+1:T}\mid z_{1:L}\bigr) \;=\;\prod_{t=L}^{T-1} p_\theta\bigl(z_{t+1}\mid \tilde{z}_{1:t}\bigr), \qquad p_\theta(\cdot\mid \tilde{z}_{1:t})=\mathrm{Categorical}\bigl(\mathrm{softmax}(u_t)\bigr), \]

where \(u_t\in\mathbb{R}^{|\mathcal{V}|}\) are logits produced by one forward pass on \(\tilde{z}_{1:t}\). The adapter passes temperature \(\tau>0\); sampling applies temperature scaling before softmax,

\[ \mathbb{P}(z_{t+1}=v\mid \tilde{z}_{1:t})\;\propto\;\exp\bigl(u_{t,v}/\tau\bigr),\qquad v\in\mathcal{V}. \]

Low \(\tau\) sharpens the distribution toward greedy-like behaviour; \(\tau\to 0^+\) approximates \(\arg\max_v u_{t,v}\) when ties are resolved per implementation.

Fig L7c — Ollama maps our max_tokens (Python) to num_predict: an upper bound on how many tokens may be generated after the prompt. Together with EOS sampling this bounds continuation length.

L7.4 · From generated text to extraction rows

Let \(Z=z_{1:T}\) be the full token sequence after stopping, and \(Y_i=\mathrm{Detok}(z_{L+1:T})\) the continuation text (assessment surface). The pipeline constrains admissible \(Y_i\) only softly via the prompt (“JSON lines only”); there is no grammar-constrained decoder in code. Thus \(\mathrm{Parse}\) from L4 is a deterministic partial map: only newline-separated substrings that decode as JSON objects contribute rows; all other lines are dropped.

\[ \mathrm{Pred}_{\mathrm{LLM}}(i)\;=\;\mathrm{Parse}\bigl(Y_i\bigr),\qquad Y_i\sim p_\theta(\cdot\mid z_{1:L}(P_i)), \]

Remark (eligibility). Unlike constrained decoding, invalid JSON does not block sampling; it yields fewer rows after \(\mathrm{Parse}\), which can trigger stub fallback (L5). The “architecture” responsible for structured assessments is therefore the joint effect of instruction tuning + greedy-ish decoding (\(\tau\approx 0.3\) default) + post-parse coercion (L4), not a separate classification head.

§1 · Setup

Fix a document index. Let \(G=(g_0,\ldots,g_{n_H-1})\) and \(L=(\ell_0,\ldots,\ell_{n_L-1})\) be finite sequences of annotation rows. Choose \(\tau\) mapping rows to match strings (entry_rus or entry_eng). Write \(T_i=\tau(g_i)\), \(U_j=\tau(\ell_j)\).

\[ \eta(s)\;:=\;\mathrm{lower}\bigl(\mathrm{strip}(\mathrm{collapse}(s))\bigr),\qquad \Sigma^\ast\to\Sigma^\ast. \]

\(\mathrm{collapse}\) replaces maximal whitespace runs by one ASCII space (Python " ".join(s.split())).

§2 · Lemma (Idempotence)

Lemma 2.1. For all \(s\in\Sigma^\ast\), \(\eta(\eta(s))=\eta(s)\).

Proof. If \(\eta(s)=\varepsilon\), stripping/collapsing stays empty. Else \(\eta(s)\) is normalized whitespace-wise; second application is idle; lower is idempotent. \(\square\)

§3 · Algorithm \(\mathcal{A}\) (content pairing)

  • Phase A. Scan \(j=0,\ldots,n_L-1\). For unused \(j\) with \(\eta(U_j)\neq\varepsilon\), pair with smallest unused \(i\) having \(\eta(T_i)=\eta(U_j)\).
  • Phase B. For unused \(i\) in ascending order, scan ascending \(j\) for substring containment with \(\min(|a|,|b|)\ge 4\) on \(a=\eta(T_i)\), \(b=\eta(U_j)\).

§4 · Theorem (Partial matching)

Theorem 4.1. Output \(P\) satisfies at-most-one partner per GT index and per LLM index; \(|P|\le\min(n_H,n_L)\).

Proof. Each pairing marks both endpoints used; guards forbid reuse. \(\square\)

Remark

\(\mathcal{A}\) is greedy and need not maximize \(|P|\).

§5 · Proposition (Exact-consistency)

Proposition 5.1. If \(\eta(T_i)=\eta(U_j)\neq\varepsilon\) and both remain unused when Phase A scans \(j\), then \(i\) pairs with some \(j'\le j\); under further hypotheses on earlier skips, \(j'=j\).

Proof. At iteration \(j\), smallest unused GT with label \(\eta(U_j)\) wins; \(i\) is eligible unless consumed earlier. \(\square\)

§6 · Content-mode accuracy

Let \(\mathcal{R}^{\mathrm{p}}=\{r:\texttt{paired\_with\_llm}=\mathrm{true}\}\), \(m=|\mathcal{R}^{\mathrm{p}}|\).

\[ \mathrm{cat}(r)=\mathbf{1}\{\texttt{human\_category}=\texttt{llm\_category}\},\quad \mathrm{fram}(r)=\mathbf{1}\{\texttt{human\_framing}=\texttt{llm\_framing}\}. \]

Theorem 6.1. For \(m>0\), \(\mathrm{Acc}^{\mathrm{cat}},\mathrm{Acc}^{\mathrm{fram}},\mathrm{Acc}^{\mathrm{both}}\in[0,100]\).

Proof. Average of indicators times \(100\). \(\square\)

Theorem 6.2. \(\mathrm{Acc}^{\mathrm{both}}\le \mathrm{Acc}^{\mathrm{cat}}\) and \(\le \mathrm{Acc}^{\mathrm{fram}}\).

Proof. Pointwise \(\mathrm{cat}\cdot\mathrm{fram}\le \mathrm{cat},\mathrm{fram}\). \(\square\)

Corollary 6.3. \(\mathrm{Acc}^{\mathrm{both}}\le \min(\mathrm{Acc}^{\mathrm{cat}},\mathrm{Acc}^{\mathrm{fram}})\).

If \(m=0\), the implementation sets all three accuracies to \(0\).

§7 · Index-mode accuracy

With \(N=\max(n_H,n_L)\), accuracies are \(100/N\) times sums of indicators over padded aligned rows — again in \([0,100]\) if \(N>0\).

§8 · \(\texttt{both\_match}\)

Proposition 8.1. \(\texttt{both\_match}\) iff \(\mathrm{cat}(r)\cdot\mathrm{fram}(r)=1\) for \(\{0,1\}\) indicators.

Proof. Code conjunction equals product of Boolean indicators. \(\square\)
